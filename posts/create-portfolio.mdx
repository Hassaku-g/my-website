## 技術スタック

スタックを選択するときは、次の 2 つのことに基づいて決定しました。

- **スピード** - 迅速に公開する必要があり、学習難易度が高く、習得に時間を要するツールの使用は避けました。
- **需要** - 実際のフロントエンド開発でよく使用されているものや今後使用が期待されるものであること

これらに当てはまる技術を選択し、最終的に Next.js、MDX、TailwindCSS、TypeScript に落ち着きました。

## Next.js

Next.js はモダンな Web サイトを構築するためのセットアップと要件を処理する React のフレームワークです。

私は次のような理由から Next.js を使用しています。
`npx create-next-app@latest`
Next.js では次のコマンドで構築するだけで Webpack、Babel、TypeScript、ESList など、簡単にセットアップできます。さらに、コード分割、画像とスクリプトの最適化、データ取得メソッドの柔軟なオプション、API ルートなどのパフォーマンス上の利点がたくさん組み込まれています。

Next.js を使用することで通常は個人開発者の範疇外にある高度な機能を実装できるようになり、その推奨事項に従うことで自然いより優れたフロントエンドアプリケーションを構築できるようになったということです。

## MDX

マークダウンファイル内で JSX コンポーネントを使用できるようになります。この機能により、記事の拡張やデモの作成が簡単に表現でき流ようになります。

私は各投稿を独自のプロジェクトとして考えています。MDX を使用すると、投稿の内容をわかりやすく説明するためん胃必要な魅力的なインタラクティブコンポーネントをすべて含めることができます。

## TailwindCSS

Tailwind は、事前定義されたクラス名を使用してコンポーネントのスタイルを設定できるユーティリティファーストの CSS フレームワークです。Tailwind は簡単に習得でき、ワークフローを大幅にスピードアップしました。Tailwind をスムーズに使い始めることができた理由はいくつかあります。

- **ドキュメントのナビゲーションが簡単** - ドキュメントの構造だけでなく、検索と例がどれほど洗練されているかもわかります。CRA または Next.js を使用して Tailwind をセットアップする方法に関するガイド
- **VS Code 拡張機能** - Tailwind CSS IntelliSense は、優れたオートコンプリートにより、ドキュメント内を参照する回数を減らしました。Headwind(サードパーティ)は、保存時にクラス名の順序を変更します。
